# お菓子なSNS

全国のお菓子好きが、写真付きで自分のお気に入りのお菓子を投稿・共有することができるSNSアプリケーションです。

**デモサイト（実際に動かせます）**
[https://okashina-sns.bolt.host/](https://okashina-sns.bolt.host/)

## 概要
「地域限定の珍しいお菓子」や「隠れた名作お菓子」の情報をストックし、共有するためのプラットフォームです。
一般的なSNSでは埋もれてしまいがちなローカルなお菓子の情報を、写真とタグ付けで整理し、後からでも探しやすくしました。

## ご利用上の注意（重要）

**1. アカウント作成について**
本アプリはポートフォリオ用のデモ環境です。セキュリティ保護のため、以下のルールを守ってください。
* **普段使用しているメールアドレス・パスワードは絶対に入力しないでください。**
* メール認証はありません。`○○○@○○○` の形式であればアカウントを登録が可能なため、架空のアドレスで登録するようにしてください。

**2. 投稿画像について**
* **お菓子の画像**のみを投稿するよう配慮をお願いします。
* 関係のない画像が投稿された場合、こちらで削除する場合があります。

### 解決したい課題
テレビやSNSで「このお菓子おいしそう！」と思っても、タイムラインの流れで情報はすぐに埋もれてしまい、「あとから探せない」「商品名を忘れてしまった」という経験がよくあります。
お菓子に特化したこのSNSでは、閲覧者が「好きなお菓子を探す楽しさ」を感じられ、投稿者も気軽に情報を発信できる場所を提供します。

## 主な機能とこだわり

* **写真・タグ付け必須の投稿**
    * テキストだけでなく、ビジュアルと検索性を重視しています。
* **「コメント機能なし」の平和な設計**
    * SNS特有の「荒れる」要素を排除するため、コメント機能はあえて実装していません。
    * 代わりに「リアクションスタンプ」を用意し、ポジティブな感情のみを共有できます。
* **地域・カテゴリー検索**
    * 旅行先のお土産選びや、自分へのご褒美探しに役立ちます。

## 技術スタック・開発手法
本アプリケーションは、生成AIを活用した現代的な開発手法「**Vibe Coding**」によって作成しました。

* **開発ツール**: [bolt.new](https://bolt.new/)
* **開発手法**: 自然言語によるプロンプトエンジニアリング（コーディングはAIが主導）
* **コードについて**: このプロジェクトはBolt Databaseを使用しているため、セキュリティの観点から接続情報を共有していません。そのため、コードをDLしただけではローカルで動かすことはできません。ご了承ください。

## 今後の展望

サービスのさらなる発展と、ユーザー体験の向上のために以下の機能を計画しています。

### 1. AIによる画像自動判定システム
現在は手動で管理している「お菓子以外の画像」の排除を、AI画像認識により自動化します。投稿された瞬間に判定を行い、運営コストを下げつつ、お菓子好きのための純粋な空間を維持します。

### 2. 店舗・企業向けアカウントとEC連携
お菓子屋さんやメーカー専用の「公式アカウント」機能を実装予定です。
* **位置情報との連携**: 近くの店舗の限定お菓子を通知。
* **直接購入機能**: 気になったお菓子をその場で購入できるEC機能を統合し、「見て楽しむ」から「食べて楽しむ」までの体験を繋ぎます。

### 3. 生成AIによる「夢のお菓子」生成機能
「こんなお菓子があったらいいな」を形にする、生成AIを活用した機能です。
ユーザーがテキストで入力したアイデアをAIがリアルな画像として生成し、タイムラインで共有できます。
* **企業との共創**: ユーザーが生成したアイデアの中から、実際のお菓子メーカーが商品化を検討するような「アイデアの宝庫」としての活用も視野に入れています。
